# Roadmap

## Sprint 0: Audit

Goal:
Understand the current product and technical state before making changes.

Outputs:
- current architecture map
- product capability audit
- product gap audit
- UX audit
- technical risk audit
- recommended implementation order

## Sprint 1: Identity Profiles

Goal:
Create persistent subject identity memory.

Build:
- subject profile
- identity notes
- reference gallery
- approved/rejected likeness examples
- do-not-change rules
- identity confidence/manual rating
- asset links to subject

## Sprint 2: Creative Recipes

Goal:
Turn successful creative workflows into reusable systems.

Build:
- recipe list
- recipe detail/create/edit
- prompt template
- negative prompt
- required inputs
- style rules
- model/provider notes
- quality checklist
- clone recipe

## Sprint 3: Generation Lineage

Goal:
Make every generation traceable.

Build:
- generation run model
- prompt/recipe/input/output tracking
- provider/model/parameter tracking
- status/errors
- rerun/duplicate flow
- generation history

## Sprint 4: Approval System

Goal:
Move review out of chaos.

Build:
- approval states
- review queue
- comments
- approve/reject/request changes
- review history
- filters by status

## Sprint 5: Asset Memory

Goal:
Make assets searchable, reusable, and connected.

Build:
- asset detail
- tags
- subject/project/campaign links
- lineage display
- search/filter system

## Sprint 6: Organization Layer

Goal:
Make the product team-ready.

Build:
- workspace structure
- projects
- campaigns/collections
- role-ready architecture

## Sprint 7: Creative Constitution UX

Goal:
Encode taste into the product.

Build:
- quality checklist
- creative direction fields
- visual language fields
- approved/rejected style references
- review warnings/checks

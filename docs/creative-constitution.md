# Pluribus Creative Constitution

## 0. Purpose

Pluribus exists to produce controlled, believable, identity-consistent, production-ready media for talent-driven industries: sports, entertainment, music, fashion, and culture-led brands. Talent scope: athletes, actors, influencers, and any human subject whose likeness requires controlled representation and legal consent.

This document defines the creative operating system of Pluribus.

It is not only an aesthetic guide.

It is a product doctrine, review system, prompt discipline, quality-control framework, and creative decision engine.

The goal is to prevent AI chaos and encode premium creative judgment into every workflow.

Pluribus should never feel like a random image generator.

It should feel like a world-class creative studio with memory.

---

# 1. Core Creative Thesis

AI generation is abundant.

Taste is scarce.

Pluribus wins by turning taste into repeatable infrastructure.

The product should not maximize visual novelty.

It should maximize:

- identity accuracy
- visual coherence
- campaign usefulness
- production readiness
- cultural credibility
- repeatable quality
- creative control
- human trust

The best Pluribus output should look intentionally directed, not algorithmically discovered.

---

# 2. Creative North Star

Every output should pass this test:

Would this asset feel credible inside a serious sports, entertainment, fashion, or media campaign?

If not, it should not be treated as production-ready.

The output must feel:

- believable
- intentional
- composed
- culturally aware
- emotionally precise
- commercially usable
- consistent with the subject
- consistent with the campaign
- free from obvious AI artifacts

---

# 3. What Pluribus Is Creatively

Pluribus is:

- a synthetic media production system
- a visual identity memory engine
- a campaign asset factory
- a creative review platform
- an art-direction system
- a consistency layer for identity-based media
- a production-grade alternative to chaotic generative workflows

Pluribus is not:

- an AI art toy
- a random prompt machine
- a visual effects slot machine
- a novelty generator
- a generic content machine
- a style-transfer gimmick
- a replacement for creative judgment

---

# 4. The Pluribus Creative Standard

A Pluribus asset must satisfy five standards:

## 1. Identity Standard

The subject must remain recognizably and consistently themselves.

No style choice is allowed to damage identity.

## 2. Direction Standard

The image must feel art-directed.

Lighting, composition, expression, framing, mood, and styling must appear intentional.

## 3. Campaign Standard

The asset must belong to a wider visual system.

It should work as part of a set, not only as a single impressive image.

## 4. Production Standard

The image must be usable in real workflows.

It should be exportable, reviewable, searchable, versioned, and tied to campaign context.

## 5. Taste Standard

The asset must avoid generic AI excess.

If it looks like “AI made this,” it fails.

---

# 5. The Hierarchy of Creative Priority

When tradeoffs happen, use this hierarchy:

1. Identity accuracy
2. Anatomical believability
3. Campaign coherence
4. Lighting and composition
5. Emotional tone
6. Wardrobe and styling accuracy
7. Background/environment quality
8. Texture and photographic realism
9. Export usefulness
10. Novelty

Novelty is last.

A boring accurate image is more valuable than a spectacular wrong one.

---

# 6. Identity Is Sacred

The subject is not raw material.

The subject is the anchor.

Pluribus must protect:

- facial structure
- eye shape
- jawline
- nose shape
- mouth shape
- skin tone
- body proportions
- hairline
- hairstyle
- beard/facial hair
- tattoos
- scars
- birthmarks
- posture
- athletic build
- recognizable expressions

The system should treat unauthorized identity changes as creative defects.

---

# 7. Identity Lock Rules

Every subject should have an Identity Profile containing:

- approved reference images
- rejected reference images
- approved generated likenesses
- rejected generated likenesses
- non-negotiable facial traits
- body/proportion notes
- hair/beard rules
- skin/tone rules
- tattoo/scar registry
- wardrobe constraints
- approved angles
- weak angles
- expression notes
- identity confidence notes

A generation should not be approved if it violates known identity constraints.

---

# 8. The Likeness Integrity Test

Before approval, ask:

- Does this still look like the subject?
- Would someone close to the subject recognize them?
- Has the face been beautified into someone else?
- Has the body been exaggerated?
- Are tattoos/scars accurate?
- Is the skin tone stable?
- Is the age consistent?
- Is the expression believable for this person?
- Does styling overpower identity?

If the image is beautiful but not accurate, it fails.

---

# 9. Anti-AI Aesthetic Rules

Pluribus must actively avoid the common AI visual disease.

Never default to:

- fake HDR glow
- waxy skin
- plastic faces
- over-sharpened pores
- glassy eyes
- random particles
- meaningless neon
- cyberpunk clichés
- floating dust for drama
- fake lens flares
- impossible rim lights
- wet metallic skin
- over-polished bodies
- symmetrical uncanny faces
- background mush
- surreal anatomy
- extra fingers
- distorted hands
- melted logos
- fake typography
- overdramatic smoke
- orange/teal grading by default
- excessive cinematic contrast
- generic “epic athlete” styling

AI should not announce itself.

---

# 10. The Restraint Principle

The highest-quality outputs are often quieter than the most impressive outputs.

Pluribus should favor:

- clean decisions
- controlled lighting
- intentional negative space
- natural skin
- believable imperfections
- simple backgrounds when useful
- precise styling
- strong silhouettes
- editorial restraint
- campaign consistency

Luxury is subtraction.

Premium work rarely screams.

---

# 11. Controlled Variation

Variation is useful only when the identity and campaign system remain stable.

Allowed variation:

- pose
- crop
- expression
- framing
- wardrobe within constraints
- controlled lighting shifts
- background options within campaign language
- export ratios

Dangerous variation:

- face drift
- age drift
- body drift
- ethnicity drift
- skin tone drift
- tattoo drift
- inconsistent hair
- random styling
- different photographic worlds in one campaign

A campaign set should feel directed by one mind.

---

# 12. Campaign Coherence

Pluribus must think in sets, not isolated images.

A campaign should define:

- visual language

- motion/energy rules

Every asset should answer:

If not, it is a draft, not a final.

# 16. Identity Strictness Levels

Each generation should define an Identity Strictness Level.

## Level 1: Loose Concept

Used for ideation only.

Identity accuracy is not production-critical.

## Level 2: Recognizable

The subject should be generally recognizable.

Useful for early campaign exploration.

## Level 3: Campaign Accurate

The subject must be clearly recognizable and suitable for review.

Default for most production work.

## Level 4: Likeness Locked

The subject must closely match approved identity references.

Required for athlete announcements, press, sponsor, and hero assets.

## Level 5: Approval Critical

No identity drift tolerated.

Required for public-facing final assets where the subject, agent, sponsor, or rights holder may review.

# 17. Output Readiness Levels

Every asset should have an Output Readiness Level.

## Draft

## Reviewable

## Production Candidate

## Approved Final

Ready for export and use.

# 19. Quality Score Framework

Pluribus should eventually support manual or automated scoring.

Suggested scoring:

## Identity Accuracy: 0 to 5

0 = wrong person  
1 = weak resemblance  
2 = partly recognizable  
3 = recognizable  
4 = strong likeness  
5 = approved likeness

## Anatomy Quality: 0 to 5

0 = unusable  
1 = major distortion  
2 = visible issues  
3 = acceptable  
4 = strong  
5 = flawless

## Campaign Fit: 0 to 5

0 = wrong campaign  
1 = off-language  
2 = partly usable  
3 = acceptable  
4 = strong fit  
5 = perfect system fit

## Production Readiness: 0 to 5

0 = unusable  
1 = concept only  
2 = needs major revision  
3 = reviewable  
4 = production candidate  
5 = final-ready

## Taste Score: 0 to 5

0 = generic AI slop  
1 = tacky  
2 = inconsistent  
3 = competent  
4 = premium  
5 = culturally sharp and production-grade

Recommended rule:

No asset can become Final unless:

- Identity Accuracy >= 4
- Anatomy Quality >= 4
- Campaign Fit >= 4
- Production Readiness >= 4
- Taste Score >= 4

# 21. Approval State Machine

Every asset should move through a clear state system:

1. Draft
2. Internal Review
3. Needs Revision
4. Production Candidate
5. Approved
6. Athlete Approved
7. Sponsor Approved
8. Final
9. Archived
10. Rejected

Final approval should require:

- identity check
- anatomy check
- campaign check
- export check
- reviewer attribution
- timestamp
- linked campaign or project
- linked subject
- linked generation run

No orphan finals.

---

# 22. Human Review Doctrine

Human review is not a bottleneck.

It is the taste layer.

The system should make review faster, more structured, and more reusable.

Reviewers should be able to say:

- approve
- reject
- request changes
- mark as likeness reference
- mark as rejected likeness
- tag failure reason
- compare against reference
- compare against campaign set
- add structured feedback

Unstructured comments are useful.

Structured feedback is compounding memory.

A strong prompt should include:

1. Subject identity constraints
2. Campaign purpose
3. visual language
4. lighting
5. camera/lens
6. composition
7. wardrobe/styling
8. background/environment

# 27. Campaign Set Review

# 33. The Rights Dimension

Every creative output produced by Pluribus is also a rights object.

It carries:

- who consented (the talent, the date, the scope)
- who approved (the creative director, the reviewer)
- what it can be used for (the licensed usage scopes)
- what it cannot be used for (revoked, restricted, or expired)
- who generated it and under what brief

The creative constitution governs what the image looks like.

The rights layer governs what the image is allowed to do in the world.

Both matter. A technically excellent image with no consent record is worthless commercially.

Pluribus must make it impossible to produce a commercially usable output without a corresponding rights record.

# 34. Final Creative Principle

Pluribus should not generate more images.

It should generate more certainty.

Certainty that the subject is right.

Certainty that the campaign is coherent.

Certainty that the asset is usable.

Certainty that the team can approve, reuse, and scale the work.

Certainty that the rights are clean.

That certainty is the product.

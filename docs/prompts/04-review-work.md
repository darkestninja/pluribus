# Review Work Prompt

Review the last completed sprint.

Read:

- docs/vision.md
- docs/current-state.md
- docs/tasks.md
- docs/decisions.md
- docs/constraints.md
- docs/handoff.md

Inspect the changed files and evaluate:

1. Does this support the Pluribus vision?
2. Does it improve workflow orchestration, identity, approval, memory, art direction, or organization?
3. Did it introduce unnecessary complexity?
4. Did it preserve existing generation flows?
5. Are there bugs or regressions?
6. Are loading/error/empty states handled?
7. Is the UX production-grade?
8. Are there missing tests?
9. Are docs updated correctly?

Output:

- Pass/fail assessment
- Issues to fix
- Suggested improvements
- Whether to commit
- Recommended commit message

Do not make changes unless explicitly asked.
